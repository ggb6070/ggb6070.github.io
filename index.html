简述搭建hexo时遇到的坑

发表于 2018-07-14 | 分类于 hexo

对于小白我来说，代码之类的都带有神秘色彩，难悟其理。
搭建博客的过程中我参考了多篇网上的教程，终于基本搭建完成，这里记录一下期间遇到的问题。
1、hexo安装失败
安装好hexo后正常，但重新打开后使用hexo命令却显示cammand not found，网上查找其解决方法，看不懂(+_+)?
无奈，怒而删，删除hexo所有的安装文件，重新安装，咦，好啦╮(╯-╰)╭问题解决
2、hexo d失败
输入hexo d命令后显示ERROR Deployer not found: git
解决方法：安装hexo-deployer-git即可，npm install hexo-deployer-git –save
哈哈轻松解决
3、主题更换失败
hexo默认的主题个人感觉不好看，更换为Next主题，成功，进行本地预览也没有发现问题，果断输入命令hexo d
把文件提交到GitHub，然而打开博客网站，主题依旧没变(＃°Д°)几经尝试依旧如此，网上没找到问题解决方法。
重看教程一遍，原来忘记使用hexo g命令，应该先使用hexo g再使用hexo d，问题解决
4、分类和关于显示失败
给hexo增添分类和关于，输入hexo new page categories和hexo new page about，然后用windows应用商店里安装
的Markdown 10进行相应.md文件的编辑，噩梦开始。。。

完全按照教程说明进行操作，就是不成功，打开分类和关于两个页面，显示：

title ：about
date:
喵喵？理论上不应该出现这些内容的，并且文章标题显示未命名，我被这个问题按在地上摩擦了一晚。
复制一份hello-world.md文件（这是安装hexo成功后自动生成的一篇文章），用Markdown 10编辑，保存，提交，查看，问题依旧存在(oﾟvﾟ)ノ
我都要怀疑这是不是hexo的bug了，最后实在不行，改其道，用Notepad++来编辑，提交，成功Σ(っ °Д °;)っ问题解决
